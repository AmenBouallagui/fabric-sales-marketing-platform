# AI Readiness

## Purpose

AI readiness means the platform's data products are documented, governed, clean, and understandable enough to support responsible AI-assisted analysis. This project treats AI readiness as a design goal rather than a final add-on.

## Readiness Criteria

- Clear domain descriptions for each curated table.
- Business-friendly column names and definitions.
- Documented metric logic and accepted filters.
- Data quality checks with visible outcomes.
- Stable keys and relationships between entities.
- Known limitations and assumptions documented.
- No sensitive real customer data in the portfolio repository.

## AI Use Cases

Potential future AI-assisted experiences include:

- Natural-language sales performance analysis.
- Campaign performance explanations.
- Pipeline risk summaries.
- Account prioritization recommendations.
- Data quality issue summaries.
- Metric documentation search.

## Governance Notes

The project uses synthetic data only. If adapted to real enterprise data, privacy, consent, retention, access control, and model grounding policies must be reviewed before enabling AI scenarios.

## Metadata Strategy

Curated tables should eventually include descriptions for tables, columns, relationships, metrics, and refresh cadence. This metadata will make the model easier for analysts and AI systems to interpret correctly.

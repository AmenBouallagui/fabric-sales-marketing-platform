# 03 Gold Modeling Notebook Plan

## Objective

Design a future Microsoft Fabric notebook that transforms valid/current Silver records into Gold dimensions and facts for Power BI reporting.

This Markdown file is a notebook implementation plan. It is not an executable `.ipynb` artifact yet.

## Parameters

Planned notebook parameters:

- `gold_processed_at`: run timestamp, generated by the notebook or passed by a pipeline.
- `start_date`: optional override for the date dimension start date.
- `end_date`: optional override for the date dimension end date.
- `write_mode`: default `overwrite` for MVP development.
- `unknown_key`: default `-1`.

## Generate Surrogate Keys

Surrogate keys should be deterministic. A hash-based key is simple for the MVP, while production designs may use controlled integer key generation.

```python
def stable_key(column_name: str):
    return F.xxhash64(F.coalesce(F.col(column_name).cast("string"), F.lit("UNKNOWN")))
```

## Handle Unknown Members

Each dimension should include an unknown row using the configured `unknown_key`.

```python
def add_unknown_member(df, values: dict):
    unknown_df = spark.createDataFrame([values])
    return unknown_df.unionByName(df, allowMissingColumns=True)
```

## Create Date Dimension

```python
def build_dim_date(start_date: str, end_date: str):
    dates = (
        spark.sql(
            f"SELECT explode(sequence(to_date('{start_date}'), to_date('{end_date}'), interval 1 day)) AS calendar_date"
        )
    )
    return (
        dates
        .withColumn("date_key", F.date_format("calendar_date", "yyyyMMdd").cast("int"))
        .withColumn("calendar_year", F.year("calendar_date"))
        .withColumn("calendar_quarter", F.quarter("calendar_date"))
        .withColumn("calendar_month", F.month("calendar_date"))
        .withColumn("month_name", F.date_format("calendar_date", "MMMM"))
        .withColumn("day_of_month", F.dayofmonth("calendar_date"))
        .withColumn("day_of_week", F.date_format("calendar_date", "EEEE"))
        .withColumn("is_weekend", F.dayofweek("calendar_date").isin([1, 7]))
        .withColumn("gold_processed_at", F.to_timestamp(F.lit(gold_processed_at)))
    )
```

## Build Facts

```python
fact_orders = (
    orders
    .join(dim_customer.select("customer_key", "customer_id"), "customer_id", "left")
    .join(dim_product.select("product_key", "product_id", "unit_cost"), "product_id", "left")
    .join(dim_campaign.select("campaign_key", "campaign_id"), "campaign_id", "left")
    .withColumn("order_date_key", F.date_format("order_date", "yyyyMMdd").cast("int"))
    .withColumn("customer_key", F.coalesce(F.col("customer_key"), F.lit(unknown_key)))
    .withColumn("product_key", F.coalesce(F.col("product_key"), F.lit(unknown_key)))
    .withColumn("campaign_key", F.coalesce(F.col("campaign_key"), F.lit(unknown_key)))
    .withColumn("net_revenue", F.col("total_amount") - F.col("tax_amount"))
    .withColumn("estimated_cost", F.col("quantity") * F.col("unit_cost"))
    .withColumn("gross_margin", F.col("net_revenue") - F.col("estimated_cost"))
    .withColumn("gold_processed_at", F.to_timestamp(F.lit(gold_processed_at)))
)
```

Equivalent fact-building logic should create `fact_ad_spend` and `fact_support_tickets`.

## Write Gold Delta Tables

```python
def write_gold_table(df, table_name: str):
    (
        df.write
        .format("delta")
        .mode(write_mode)
        .option("overwriteSchema", "true")
        .saveAsTable(table_name)
    )
```
